# Card-Launch (p7hb) — How It's Animated

Turns the static `assets/animate-illustration-attached-p7hb.svg` into a one-shot
"rocket launch" entry at
`public/projects/animate-illustration-attached-p7hb/scene-1/lottie.json`,
generated by `scripts/build-animate-illustration-attached-p7hb.mjs`. Like every
scene here it's a build script, not hand-authored JSON: it parses the SVG path
data and emits every shape/keyframe programmatically. Re-run
`node scripts/build-animate-illustration-attached-p7hb.mjs` to regenerate.

## What the source actually is

`qlmanage`-rendering the source first (the fastest way to read an unfamiliar
illustration — `qlmanage -t -s 512 -o /tmp assets/<file>.svg`) revealed the
"card" is really a **rocket-shaped card**: a white trapezoid with a bold black
spine (`Fill 19`), already drawn mid-launch. Below it sit exhaust **motion
lines** (`Stroke 11/13` a broken vertical centre line, `Stroke 15/17` two curves
fanning out), a wide **cloud** (`Fill 3` + `Stroke 5`), a faint **diagonal-hatch
dome** backdrop (`Fill 1`, a raster pattern at 15% opacity), and four decorative
**dots** (`Fill 21/23/25/27`). The brief ("card appears in centre, takes off
like a rocket, cloud appears as it rises, cloud fades at the end") maps one-to-one
onto these parts — recognising the illustration is *already* a launch tableau is
what makes the choreography obvious.

## Choreography (60fps, 180f = 3s, plays once)

| beat | frames | what happens |
| --- | --- | --- |
| APPEAR | 0–38 | card fades + scales in at a low-centre "ready" spot, settles; dome fades in; dots drift + fade in on staggered offsets |
| ANTICIPATE | 38–46 | small downward crouch + squash (wider/shorter) — loading energy |
| LAUNCH | 46–100 | card rockets up to its source position with a vertical stretch peaking mid-flight + small overshoot; exhaust lines stream on (trim draw); cloud billows in beneath |
| FLOAT | 100–135 | card holds high with a buoyant bob; dots keep drifting |
| DISSIPATE | 135–180 | cloud fades away + gently expands (smoke dissipating); card stays up; dots settle to source |

**The final frame equals the source composition MINUS the cloud.** The generic
ENTRY note ("settle exactly on the source") is overridden here by the brief's
explicit "the cloud fades away at the end" — every *other* element lands on its
exact source geometry, so the settle is still a real composition, just with the
smoke gone.

## The one bug that bit, and the rule it confirms

**Within a shape layer's `shapes` array, index 0 paints FRONTMOST.** First build
put the rocket as `[cardGroup(white fill), stripeGroup(black)]` and the black
spine vanished — the white card body (index 0) was painting *over* the spine.
Swapping to `[stripeGroup, cardGroup]` fixed it. This is the same ordering as the
top-level `layers` array (index 0 frontmost) and as items inside a `group`'s `it`
(earlier = in front). When a fill is hiding something that should sit on top of
it, it's almost always this — move the on-top shape earlier in the array.

## Splitting position and scale across two nulls (the freeze-bug dodge)

The rocket must both **rise** (animated position) and **squash/stretch** about
its base (scale with a non-zero pivot). Doing both on one layer is the known
freeze combo (non-zero anchor + animated position). So it's two chained nulls:

- `mover` — animates position only, anchor `[0,0,0]` (the safe animated-position
  combo). Carries APPEAR→CROUCH→LAUNCH-overshoot→settle→bob as a `[0, Δy, 0]`
  offset.
- `scaler` (parent = mover) — anchor + static position both `= [cardCx, cardBaseY]`
  (`BASE`), animated scale only (safe combo). Pivoting at the **base** makes the
  launch stretch read as upward thrust, not a centred zoom.
- `rocket` shape layer (parent = scaler) — zero transform, just the opacity fade.
  (Opacity is *not* inherited from a parent null, so it lives on the shape layer.)

## Exhaust: trim-path draw in the direction you want

The four exhaust strokes "stream on" via a `ty:'tm'` trim modifier keyframing
`e` 0→100 (round caps, `lc:2`, so the growing tip reads like motion). Two of them
(`Stroke 15/17`) are authored bottom→top, but the exhaust should draw *from the
rocket base downward*, so they're run through `reverseSubpath` (reverse vertices
**and** swap in/out tangents) before trimming. The centre line leads, curves
follow +4f, the lower dash trails — a small stagger so they don't switch on in
lockstep. They're timed to start just after the card ignites (t52+) and finish
by t92, so the draw-on visibly belongs to the launch.

## Cloud: billow + dissipate on the safe transform path

One layer, two groups — a white **evenodd** fill (path forced `c:true` so the
open source path closes across the bottom, matching its implicit fill) and the
open scalloped **outline** stroke. Animated with own-centre anchor + **scale**
(55%→105%→100% billow, then →120% expand on the way out) and **opacity**
(0→100→0). No position animation — pure scale+opacity keeps it on the safe combo
and reads as "puffs in, then dissipates."

## Dome hatch: raster pattern → real strokes + track matte

The source dome is a raster hatch at 15% opacity. Reproduced faithfully without
the image: a fan of diagonal `/` strokes (`hatchLines`, spacing 7) is clipped to
the dome silhouette with a **track matte** — a white-filled dome path layer
(`td:1`) immediately above the hatch layer (`tt:1`), the same pattern
`build-dataprocessing.mjs` uses. The matte and matted layers **must stay adjacent
in the `layers` array** (matte first / lower index), so the non-rendering
`mover`/`scaler` nulls are placed earlier and the dome pair is last. Faintness is
carried by the hatch layer's opacity (12), fading in during APPEAR.

## Dots: fade + gentle drift, settling to source

Each dot is one layer, anchor `[0,0,0]`, animating **position** (a small ±3–7px
drift on a per-dot phase, returning to `[0,0]` by t180 so they land on source
geometry) + **opacity** (staggered fade-in t8–t40). No scale, so they stay on the
safe animated-position combo. "Moving around" is deliberately subtle — restraint
reads more premium than orbiting.

## Verification

Rendered through `scripts/preview-scene.mjs` (deterministic CanvasKit-in-Node):
frame 0 (near-empty entry start), the appear/crouch beats, the launch + billow +
exhaust draw, the dissipate, and f179 zoomed ×2 against the `qlmanage` source —
confirming the settled frame matches the source composition minus the faded
cloud, with the spine, exhaust, dome and dots all on their exact source geometry.

## Applying this elsewhere

1. `qlmanage`-render an unfamiliar SVG first — knowing it's a rocket-launch
   tableau reframed the whole brief.
2. If a fill hides a shape that should be on top, reorder — index 0 is frontmost
   in `shapes`, `it`, and `layers` alike.
3. Rise + pivot-scale on one element → split across a position null (zero anchor)
   and a scale null (non-zero pivot, static position). Never both on one layer.
4. Trim-draw stationary line art (`tm`, `e` 0→100, round caps); `reverseSubpath`
   any stroke authored against the direction you want it to grow.
5. Reproduce a raster fill pattern as real strokes + a track matte; keep the
   matte/matted pair adjacent (matte first) and nulls out from between them.
